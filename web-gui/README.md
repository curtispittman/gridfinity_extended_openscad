# Gridfinity Extended — self-hosted web GUI

A browser-based generator for this repo's models, similar to the online Gridfinity
generators: pick a model, tweak its parameters in a form, preview it in 3D, and
export an STL. It runs [seasick/openscad-web-gui](https://github.com/seasick/openscad-web-gui)
(OpenSCAD compiled to WebAssembly) patched for this project.

**All rendering happens in the visitor's browser.** The container just serves static
files, so it's light on the NAS — but render speed depends on the client device.

## Deploy on a NAS with Portainer

1. Portainer → **Stacks** → **Add stack** → **Repository**.
2. Repository URL: your fork of this repo. Compose path: `web-gui/docker-compose.yml`.
3. **Deploy**. Portainer clones the repo, builds the image (clones openscad-web-gui,
   downloads the OpenSCAD WASM build, patches and bundles it), and starts nginx.
4. Open **`http://<nas-host>:8088/`**.

Change the published port by editing `ports:` in `web-gui/docker-compose.yml`
(default `8088:80`).

## Deploy locally / over SSH

```bash
docker compose -f web-gui/docker-compose.yml up -d --build
# open http://localhost:8088/
```

## Using it

- The landing page (`/`) lists every model in `combined/`. Click one to open it in
  the **Customizer**.
- Adjust parameters, watch the 3D preview update, then **Export STL**.
- Deep links work too: `http://<host>:8088/?http://<host>:8088/models/gridfinity_basic_cup.scad`.

## How it works

- Serves the self-contained files from [`combined/`](../combined) at `/models/`
  (these inline all `modules/<...>` includes, so they load cleanly into the WASM
  filesystem — the root `.scad` files would not).
- `gallery.html` enumerates `/models/` via nginx's JSON autoindex, so new models
  appear automatically after a rebuild — no manifest to maintain.
- `nginx.conf` serves the picker at `/` when there's no query string and the
  OpenSCAD app when deep-linked.

## What `patch.mjs` changes (and why)

Verified against pinned commit `759feee` of openscad-web-gui with the
OpenSCAD `2025.03.25` WebAssembly build:

| Patch | Why |
|-------|-----|
| `--backend=manifold` | Uses the Manifold backend. **Verified ~0.25 s vs ~31 s** per bin — without it, full renders fall back to CGAL and the tool is unusably slow. |
| `--enable=textmetrics` | This repo's label/`Text` module calls `textmetrics()`. Without the flag OpenSCAD warns *"Ignoring unknown function 'textmetrics'"* and text is mis-sized. The bundled WASM build supports it. |
| Empty CORS-proxy default (`esbuild.app.mjs`) | Makes the app fetch model files directly (same-origin) instead of via the upstream's `localhost:8888` dev proxy, so no proxy service is needed. |

The patch asserts every edit; if upstream changes shape the Docker build fails
loudly instead of shipping an unpatched app.

## Updating

- **New/changed models:** they live in `combined/` and are baked into the image —
  re-deploy the stack (rebuild) to pick them up.
- **WASM build:** the Dockerfile auto-discovers the newest `*-WebAssembly-web.zip`
  from <https://files.openscad.org/playground/>, falling back to the pinned
  `WASM_FALLBACK` in [`Dockerfile`](Dockerfile). If both ever break, update that ARG.

## Caveats

- **Client-side compute.** Heavy models (large baseplates, dense voronoi, high
  `$fn`) render slower and use more memory in the browser; very large models can
  hit the tab's WASM memory ceiling. Weak clients (phones/tablets) feel this most.
- **Fonts.** The bundle ships Liberation Sans. This repo's label fonts (Aldo,
  Bahnschrift, …) aren't included, so OpenSCAD substitutes the default font —
  geometry is correct, exact label glyphs may differ from a desktop render.
- **Customizer widgets.** The form is generated by openscad-web-gui's parser, which
  is close to but not identical to desktop OpenSCAD's Customizer; an occasional
  control may render as a plain field.
